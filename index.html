My Project Idea. (or how to change the world)



Overview:
Create a simple BINGO game. My machine will in it’s simplest form will be a random number generator. But it will have a few of buttons,
screen(s) and LED’s to display other data and machine status.
Further, the machine will be able to display previously chosen numbers to check the correctness of the bingo card.
When a random number is selected a display will show the nickname of the number or some interesting fact or tradition about the number.
For example “Legs Eleven”. The machine will also have a primitive menu.

Motivation:
A lot of clubs run Bingo Nights using old electro-mechanical machines containing up to 90 table tennis balls each printed with a unique number.
The machine uses suction to draw the ball out of the machine. Before any Bingo Night quite a few minutes are spent checking that all the balls are present.
Over time the balls get lost or trodden on. This problem is often discovered 5 minutes before the game starts and its often too late.
The machine gets worn and the suction of the balls out of the machine becomes strained. It has a motor that often needs repairs.
Another problem is that the Bingo machines usually need an extension cord and these spread out on the floor, which in these days of OH&S is not acceptable.
Also the machines are often very noisey.

Description:
My machine will be based around an Arduino board which can be powered by a USB connector, or by a 9V Battery through a barrel jack.
No more extension cords on the floor. The machine will have two modes; Game Mode and Checking Mode.
Pressing a button will switch the machine between these two modes.



Game Mode
The size of a bingo card in Australia is 1 to 90, so the software will select a random number between these values (inclusive).
Initially these maximum and minimum numbers will be specified in the program. Pressing the new number button will cause the software to choose a random number using Arduino’s random() function.
It checks to see if that number has already been chosen. If it has, it will select a new number,
If it hasn’t it will display the new number on an LED display and record that the number has now been chosen.
It will also increase and display the cumulative numbers selected. Knowing how many numbers have been called or selected is important because bonuses are sometimes paid if a winner achieves a winning card in a nominated number of numbers.
The machine will also display various nicknames associated with Bingo numbers like “Legs 11” or “Cup of Tea” (Number 3), or if no nickname exists an interesting fact about that number.
These facts will initially be contained withing the program. This will give the ‘caller’ something to talk about while players mark their card.
The process of selecting random numbers will repeat until a player has a complete card. The operator then switches the machine to ‘Checking Mode’

Checking Mode.
All numbers on the winning card are entered via the keypad The machine will check each number to see if has been ‘called’.
If it hasn’t been called then the card is incorrect and the machine can be switched back to ‘Game Mode’ and normal game play can continue.
If the card is correct a new game can be started.



Additionally the machine will have a dice mode. That is, it will select a random number between one and 6(inclusive).
The operator will have the option of simulating one or two die. This can be used as a fun way to play Backgammon or board games like Monopoly.
For one thing dice will not go all over the place after being thrown.

Tools and Technologies
The machine is an Arduino board communicating with a number of sensors. The “Menu” display will be on an LCD 1602. (2 lines of 16 characters each).
The random number(s) will be displayed on 4 digit 7 segment display. While the machine is calculating a random number the progress will be shown on an LED Bar Graph.
Buttons, buzzers and LED’s will display the machine status. A Keypad will be used for entering numbers during the checking mode. The final product will be housed in a self contain box.
Software will be written from scratch but should consist of 2 parts a) Random Number generation (Bingo and Die) and b) Sensor Control.
Code for handling of sensors are readily available on the net so most of the programming will be concerned with the random number generation and recording.
As an option the machine could be sold in 2 versions. The base model described above and the deluxe model with an Infrared Remote to could control operations.
Skills Required:
The two principal skills required during the manufacturing phase are a) an understanding of electronics and b) computer programming skills.

An understanding of electronics
The first part of the project will be the schematic design and being able to draw this is essential. Knowledge of the components and their limitations is also important.
Once the schematic is complete, prototyping then going straight to board design and building is the next step. Reading the schematic and soldering are the dominant skill

Computer programming skills.
Initially writing or designing an algorithm then writing the Arduino program(s) (or sketches). These programs are written in C++.
So knowledge of the Object-oriented programming is essential
Other skills would include Writing of Documentation (Operators Manual etc) and testing of the machine. These 2 would normally be in an iterative process.
After manufacture it’s up to the Sales department to determining what the market will pay and then sell the machine for a profit.
